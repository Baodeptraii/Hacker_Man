Weak Session IDs là gì ?

Weak Session IDs - việc sử dụng ID phiên yếu là một trong những lỗ hổng bảo mật phổ biến nhất trong các ứng dụng web. Khi một ứng dụng web sử dụng ID phiên yếu, kẻ tấn công có thể dễ dàng chiếm quyền điều khiển phiên của người dùng, và thực hiện các hành động độc hại như thay đổi thông tin cá nhân, thực hiện các giao dịch không hợp lệ, hoặc thậm chí chiếm quyền kiểm soát toàn bộ hệ thống.

Ae cần đảm bảo ID phiên của mình là không thể đoán được, nếu không kẻ tấn công có thể chiếm đoạt được ID phiên bằng những tập lệnh tương đối đơn giản. Hầu hết các framework ngày nay đều cung cấp cách tạo ID phiên mạnh mẽ, tốt nhất là ae không nên tự chế ra framework của mình :v

ID phiên cần được chọn từ một không gian địa chỉ lớn (tức là không thể đoán được), và cần được tạo ra một cách ngẫu nhiên bằng cách sử dụng một thuật toán mạnh mẽ.


Cách để ngăn chặn Weak Session IDs

Use Built-In Session Management (Sử dụng quản lý phiên tích hợp)

Sử dụng các công cụ quản lý phiên tích hợp trong các framework web phổ biến như ASP.NET, PHP, và Java. Điều này giúp đảm bảo rằng ID phiên được tạo ra một cách an toàn và không thể đoán được. Một số code mẫu ở bên dưới sẽ trình bày một số cách tạo ID phiên đủ an toàn cho ae.

Tamper-Proof Your Cookies (Bảo vệ Cookies của bạn)

Đảm bảo rằng cookies chứa ID phiên được mã hóa và ký số để ngăn chặn kẻ tấn công sửa đổi nó. Điều này giúp ngăn chặn kẻ tấn công sử dụng ID phiên của họ để chiếm quyền điều khiển phiên của người dùng.

Code tham khảo

Python - Django 
def get_random_string(length=12,
    allowed_chars='abcdefghijklmnopqrstuvwxyzABCDEFGHIJKLMNOPQRSTUVWXYZ0123456789'):
    return  ''.join(random.choice(allowed_chars) for _ in  range(length))
Ruby - Rails 
def generate_session_token
    loop  do
        token = SecureRandom.urlsafe_base64
        return  token unless  User.exists?(session_token: token)
    end
end
Node - Express.js 
const crypto = require('crypto');
const token = crypto.randomBytes(64).toString('hex');
