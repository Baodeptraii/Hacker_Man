Downgrade Attacks là gì ?

Downgrade Attacks (Tấn công hạ cấp) xảy ra khi kẻ tấn công buộc một hệ thống phải sử dụng phiên bản cũ hoặc kém an toàn hơn của một giao thức, phần mềm, hoặc thuật toán mã hóa. Điều này làm suy yếu các biện pháp bảo mật và khai thác các lỗ hổng đã được vá trong các phiên bản mới hơn.

Lưu lượng truy cập Internet được bảo vệ bởi Transport Layer Security (TLS). Các trang web bảo mật sử dụng HTTPS, tức là giao thức HTTP được truyền qua TLS (hoặc tiền thân của nó, Secure Sockets Layer - SSL). Qua từng năm, TLS đã dần cải thiện bảo mật, với các phiên bản mới hơn thường vá các lỗ hổng bảo mật của các phiên bản cũ hơn. Tuy nhiên, nếu một trang web không hỗ trợ phiên bản TLS mới nhất, kẻ tấn công có thể buộc trang web sử dụng phiên bản cũ hơn, dễ bị tấn công hơn.

Cách ngăn chặn Downgrade Attacks

Specify a Minimum Version of TLS (Sử dụng phiên bản đủ mạnh)

Để giảm thiểu các cuộc tấn công hạ cấp, máy chủ web của ae nên được cấu hình để sử dụng phiên bản TLS mạnh tối thiểu . Phiên bản TLS tối thiểu hiện tại được khuyến nghị là TLS 1.3.

Việc chỉ định phiên bản TLS tối thiểu sẽ không gây gánh nặng quá mức cho hầu hết các ứng dụng web, vì các trình duyệt hiện đại có khả năng tự cập nhật và thường sử dụng các tiêu chuẩn mã hóa mới nhất. Tuy nhiên, một số loại ứng dụng web không thể quá nghiêm ngặt trong cách tiếp cận của chúng: nếu ae đang duy trì các dịch vụ web cho các thiết bị nhúng, thì các máy khách như vậy hiếm khi nhận được các bản cập nhật bảo mật, vì vậy, thật không may, ae sẽ phải sử dụng các tiêu chuẩn mã hóa cũ hơn trong thời gian khá dài.

Ensure All Traffic is Sent Over HTTPS (Đảm bảo tất tả lưu lượng chỉ truyền qua giao thức HTTPS)

Để đảm bảo mọi lưu lượng truy cập đến trang web của ae được gửi qua kết nối an toàn, ae nên cấu hình máy chủ web của mình để chuyển hướng mọi kết nối không an toàn (cổng 80) đến đnơi an toàn hơn (cổng 443).

Ae cũng nên triển khai tiêu đề HTTP Strict Transport Security (HSTS) để cho trình duyệt biết rằng chúng chỉ nên tạo kết nối an toàn đến máy chủ web của ae. 

Strict-Transport-Security: max-age=31536000

Code tham khảo

Máy chủ web Nginx 
server {
    listen 80;
    server_name example.com;

    # Chuyển từ HTTP sang HTTPS
    return 301 https://$server_name$request_uri;
}

server {
    listen 443 ssl; server_name example.com;

    # Sử dụng chứng chỉ số để mã hóa, 
    # và một cặp khóa bí mật để giải mã.
    ssl_certificate /path/to/ssl/certificate.crt;
    ssl_certificate_key /path/to/ssl/private.key;

    add_header Strict-Transport-Security "max-age=31536000";

    # Sử dụng phiên bản TLS đủ mạnh
    ssl_protocols TLSv1.3;
}
Máy chủ web Apache 
# Chuyển từ HTTP sang HTTPS
<VirtualHost *:80>
    ServerName example.com
    
    RewriteEngine On
    RewriteCond %{HTTPS} off
    RewriteRule ^ https://%{HTTP_HOST}%{REQUEST_URI} [L,R=301]
</VirtualHost>

# Tải SSL module
LoadModule ssl_module modules/mod_ssl.so

# Cấu hình SSL/TLS 
<VirtualHost _default_:443>
    ServerName example.com

    Header always set Strict-Transport-Security "max-age=31536000"

    # SSL Engine 
    SSLEngine on

    # Sử dụng chứng chỉ số để mã hóa, 
    # và một cặp khóa bí mật để giải mã.
    SSLCertificateFile /path/to/your/certificate.crt
    SSLCertificateKeyFile /path/to/your/private-key.key
    
    # Sử dụng phiên bản đủ mạnh
    SSLProtocol +TLSv1.3

    
    # Logging
    ErrorLog "/var/log/httpd/error_log"
    TransferLog "/var/log/httpd/access_log"
</VirtualHost>
